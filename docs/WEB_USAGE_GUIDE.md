# Web 界面使用说明

## 1. 系统简介

本项目提供了一个轻量级 Web 原型，用于演示“基于深度哈希的网络入侵检测日志去重系统”的核心流程。  
前端采用 `Vue 3 + Vite + Element Plus`，后端采用 `Spring Boot + H2`，并通过 Python 查询能力完成去重结果演示。

这个 Web 原型适合用于：

- 课程设计或毕业设计演示
- 本地联调与功能验证
- 论文中的页面展示与系统说明

## 2. 访问地址说明

系统启动后，常用入口如下：

- 前端业务系统登录页：`http://127.0.0.1:5173/`
- 后端服务说明页：`http://127.0.0.1:8080/`
- 后端接口示例：`http://127.0.0.1:8080/api/stats/overview`
- H2 控制台：`http://127.0.0.1:8080/h2-console`

### 2.1 什么是前端业务系统

前端业务系统就是用户真正操作的页面，包括：

- 登录页
- 首页仪表盘
- 日志导入
- 参数配置
- 任务执行
- 结果展示
- 统计分析

如果你要“登录系统”，请始终访问：

`http://127.0.0.1:5173/`

### 2.2 什么是后端接口示例

后端接口示例是给前端程序调用的数据接口，不是给用户直接操作的业务页面。  
例如：

`http://127.0.0.1:8080/api/stats/overview`

浏览器打开后看到的是一段 JSON 数据，这是正常现象。它说明后端接口工作正常，前端页面就是通过这些接口来读取任务数、压缩率、攻击类型分布等统计信息。

### 2.3 什么是 H2 控制台

H2 控制台是开发期数据库管理页面，用于：

- 查看表结构
- 检查种子数据
- 执行 SQL
- 辅助调试

它不是系统业务登录页，也不是给普通使用者操作的页面。

如果需要连接本项目数据库，请使用：

- JDBC URL：`jdbc:h2:file:./data/logdedup`
- User Name：`sa`
- Password：留空

## 4. 登录系统

### 4.1 登录步骤

1. 打开 `http://127.0.0.1:5173/`
2. 输入用户名
3. 输入密码
4. 选择对应角色
5. 点击“登录系统”

登录成功后，会自动进入首页仪表盘。

### 4.2 登录失败时优先检查

- 是否访问的是 `5173` 前端地址，而不是 `8080`
- 用户名、密码、角色是否对应
- 后端 `8080` 服务是否已经启动
- 前端页面是否是旧缓存版本

浏览器如果仍显示旧页面，建议强制刷新：

- macOS：`Cmd + Shift + R`

## 5. 页面功能说明

## 5.1 首页

首页用于查看系统总体运行情况，主要展示：

- 任务总数
- 日志总量
- 平均压缩率
- 平均时延
- 最近任务列表
- 攻击类型分布图
- 平台运行概览图

适合在演示时先展示系统总体效果。

## 5.2 日志导入

“日志导入”页面用于把 CSV 文件上传到某个已有任务中，并查看预览结果。

使用步骤：

1. 先在“参数配置”页面创建任务
2. 进入“日志导入”
3. 选择关联任务
4. 选择 `.csv` 文件
5. 点击“上传并预览”

上传成功后，页面会显示：

- 文件名
- 任务编号
- 总行数
- 上传结果消息
- 前几行预览数据

## 5.3 参数配置

“参数配置”页面用于创建一个新的去重任务。

可配置参数包括：

- 任务名称
- 相似度阈值
- 时间窗口（分钟）
- 保留策略
- 哈希编码长度
- 查询样本 `rowIndex`
- 标签范围 `labelScope`
- 返回候选数 `topK`

其中：

- `similarityThreshold` 表示去重时的相似度阈值
- `timeWindow` 表示聚合同一时间范围内日志的窗口大小
- `reservePolicy` 表示保留最早、最新或高风险日志
- `hashBits` 表示哈希编码长度，当前支持 `32` 位和 `64` 位

点击“保存任务参数”后，系统会创建一个新任务。

## 5.4 任务执行

“任务执行”页面用于查看任务状态并启动任务。

页面中可以看到：

- 任务编号
- 任务名称
- 状态
- 阈值
- 压缩率
- 时延
- 操作按钮

你可以点击某一行的“启动”，触发后端执行去重任务。

状态说明：

- `PENDING`：待执行
- `RUNNING`：执行中
- `SUCCESS`：执行成功
- `FAILED`：执行失败

页面下方还会展示：

- 待执行/运行中任务数量
- 已完成任务数量
- 最近一次运行消息

## 5.5 训练演示（论文复现 + 本地训练）

“训练演示”页面用于用图形化方式触发本地训练流程，并在页面中查看训练日志（tail）。

当前支持两类训练：

- 论文复现（DeepLSH / Stack Traces）：选择 `Measure`、`n`、`epochs` 等参数，点击“创建并启动”
- CIC-IDS-2017 本地训练：按顺序点击 `Prepare -> Train MLP -> Train Bi-GRU -> Eval`

提示：

- 页面会自动轮询 `RUNNING` 的训练任务状态与日志
- 训练产物会落在项目目录（例如 `artifacts/stacktraces/`、`artifacts/cicids/`）

## 5.6 结果展示

“结果展示”页面用于查看某个任务的去重结果。

支持功能：

- 按任务筛选
- 刷新结果
- 查看结果详情

表格字段包括：

- 编号
- 日志 ID
- 攻击类型
- 相似度
- 聚类编号
- 冗余标识

点击“详情”后，会在右侧抽屉中显示：

- 候选样本编号
- 来源文件
- 哈希编码
- 代表日志
- 聚类编号

其中“冗余标识”含义如下：

- `冗余`：该日志被判定为冗余日志
- `保留`：该日志被保留为代表日志

## 5.7 统计分析

“统计分析”页面主要用于展示可视化结果，适合答辩时展示系统效果。

当前包含：

- 压缩率柱状图
- 攻击类型饼图
- 任务时延折线图
- 冗余趋势图

这些图表的数据来自后端统计接口 `/api/stats/overview`。

## 6. 推荐演示流程

如果你要做课堂展示、毕业设计答辩或录屏，推荐使用下面的顺序：

1. 打开登录页并登录
2. 进入首页，展示任务总数、压缩率和攻击类型分布
3. 进入“训练演示”，展示一键触发训练与日志输出（可用 smoke 预设快速演示）
4. 打开“参数配置”，创建一个新任务
5. 进入“任务执行”，启动刚创建的任务
6. 进入“结果展示”，查看去重结果和详情
7. 进入“统计分析”，展示图表
8. 如有需要，再进入“日志导入”展示 CSV 上传和预览

## 7. 常见问题

### 7.1 访问 `8080` 时看到的不是登录页

这是正常现象。  
`8080` 是后端服务端口，不是前端登录页。  
真正的登录页在：

`http://127.0.0.1:5173/`

### 7.2 访问 `/api/...` 时看到一段 JSON

这是正常现象。  
说明你访问的是后端接口，而不是前端页面。

### 7.3 打开 H2 控制台后看到数据库登录页

这也是正常现象。  
H2 控制台是数据库管理工具，不是业务系统登录页。

### 7.4 登录失败

请检查：

- 用户名是否正确
- 密码是否正确
- 角色是否选择正确
- 后端 `8080` 是否已启动
- 前端是否已刷新到最新版本

### 7.5 任务执行后没有看到结果

请检查：

- 任务是否执行成功
- 是否在“结果展示”中选择了正确任务
- 当前 Python 查询链路是否可用

如果模型文件或完整运行环境不可用，系统会优先保证演示闭环，但结果内容可能以演示数据或回退逻辑为主。

## 8. 开发与调试建议

如果你需要继续开发或排查问题，建议同时关注以下三个入口：

- 前端页面：`http://127.0.0.1:5173/`
- 后端说明页：`http://127.0.0.1:8080/`
- H2 控制台：`http://127.0.0.1:8080/h2-console`

这样可以快速区分：

- 是前端显示问题
- 是后端接口问题
- 还是数据库数据问题

## 9. 总结

这套 Web 原型已经覆盖了一个完整的演示闭环：

- 登录
- 参数配置
- 任务创建
- 任务执行
- 结果展示
- 统计分析
- 日志导入

其中：

- `5173` 负责前端业务页面
- `8080/api/...` 负责后端接口
- `8080/h2-console` 负责开发期数据库调试

在使用时，只要记住“登录看 5173，接口看 8080/api，数据库看 h2-console”，就不容易混淆。
