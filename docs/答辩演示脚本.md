# 毕设答辩演示脚本（8-12 分钟）

## 0. 开场（30s）
- 题目：基于深度哈希的网络入侵检测日志去重系统
- 问题：安全告警存在大量近似重复，人工研判压力大、存储开销高
- 目标：做一套可运行、可展示、可复现的轻量 Web 原型

## 1. 问题与思路（1.5 min）
- 传统规则匹配对模板变体和轻微字段波动不鲁棒
- 核心思路：日志预处理 + 深度编码 + 二值哈希 + 近邻检索
- 价值：兼顾检索效率与近似重复识别能力

## 2. 系统架构（1.5 min）
- 前端：Vue3 + Element Plus
- 后端：Spring Boot
- 算法：Python `code/run.py` 管线
- 数据层：H2（开发默认，可扩展 MySQL）
- 演示要点：前后端解耦，算法可替换

## 3. 功能演示（3-4 min）
1. 登录系统（管理员或运维）
2. 参数配置：阈值、时间窗口、保留策略、哈希位数
3. 任务执行：观察状态从 `PENDING -> RUNNING -> SUCCESS`
4. 结果展示：筛选任务，查看冗余标识与聚类详情
5. 统计分析：查看压缩率、攻击类型分布、时延趋势

## 4. 实验结果（2 min）
- 数据口径：CIC-IDS-2017，`n_flows=12000`，`n_pairs=20000`
- 最优模型：`bigru-deeplsh`
- 指标：Accuracy 83.16%，Precision 78.34%，Recall 91.64%，F1 84.47%
- 压缩率：83.08%
- 平均 Top-K 查询时延：111.91 ms
- 结论：可在可接受时延下实现较高召回与压缩效果

## 5. 不足与改进（1 min）
- 目前为验证性原型，尚未覆盖实时流式接入与企业级权限体系
- 未来方向：
  - 引入更强语义模型
  - 融合时间序列上下文降低误判
  - 多源设备与实时告警收敛联动

## 6. 结束语（20s）
- 本系统已形成“方法 + 系统 + 实验”闭环
- 可作为入侵检测日志治理的工程化起点
